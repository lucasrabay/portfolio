Most RAG tutorials end where production begins. They show you how to split documents, embed them, and retrieve top-k results. What they skip is everything that breaks when real users show up with messy queries and expect instant responses.

## The Chunking Problem

The first thing that failed in our pipeline was chunking. We started with naive fixed-size splits at 512 tokens. This worked for clean markdown documentation but destroyed the structure of academic papers, where a single argument might span three paragraphs.

We moved to semantic chunking using sentence-level embeddings and a sliding-window similarity threshold. When the cosine similarity between consecutive sentences dropped below 0.72, we cut a new chunk. This preserved logical boundaries without requiring format-specific parsers.

```python
def semantic_chunk(sentences: list[str], threshold: float = 0.72) -> list[list[str]]:
    embeddings = model.encode(sentences)
    chunks, current = [], [sentences[0]]

    for i in range(1, len(sentences)):
        sim = cosine_similarity(embeddings[i-1], embeddings[i])
        if sim < threshold:
            chunks.append(current)
            current = [sentences[i]]
        else:
            current.append(sentences[i])

    chunks.append(current)
    return chunks
```

## Embedding Selection Matters More Than You Think

We benchmarked five embedding models on our domain-specific retrieval task. The winner was not the largest model. A fine-tuned `bge-base-en-v1.5` outperformed `text-embedding-3-large` on our academic corpus by 12% on recall@10, while being 4x cheaper to run.

The lesson: always evaluate on your data. General benchmarks (MTEB, BEIR) are useful starting points, but domain-specific relevance can diverge dramatically from aggregate scores.

## Latency Is the Real Boss

Our first deployment had a p95 latency of 3.2 seconds. Users abandoned queries after 2 seconds. We got it under 800ms with three changes:

1. **Pre-computed HNSW index** instead of brute-force similarity search
2. **Streaming completions** so the first token appears within 200ms
3. **Query caching** with a 5-minute TTL for repeated academic concepts

Retrieval-augmented generation is not retrieval plus generation. It is an engineering discipline with its own failure modes, performance characteristics, and design trade-offs. Treat it like infrastructure, not a demo.
